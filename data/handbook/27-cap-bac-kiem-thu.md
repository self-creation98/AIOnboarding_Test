# Cấp bậc Kiểm thử (QA)

> Tài liệu nội bộ Company. Cập nhật: 04/2026.

## Junior QA Engineer (L2)

**Thâm niên tham khảo**: 0-2 năm.

- Thực hiện test case có sẵn (manual testing).
- Viết bug report rõ ràng: steps to reproduce, expected vs actual, screenshot/video.
- Hiểu cơ bản về test types: functional, regression, smoke, sanity.
- Tham gia test trên nhiều thiết bị và trình duyệt.
- Sử dụng được Jira để quản lý test case và bug tracking.
- Học viết automation test cơ bản (Selenium, Cypress, hoặc Playwright).

## Mid-level QA Engineer (L3)

**Thâm niên tham khảo**: 2-4 năm.

- Tự viết test plan và test case cho feature mới từ PRD/spec.
- Viết automation test cho regression suite (coverage > 60%).
- Thực hiện API testing (Postman, RestAssured).
- Tham gia vào sprint planning: estimate effort test, identify risks.
- Hiểu CI/CD pipeline và integrate automation test vào pipeline.
- Performance testing cơ bản (JMeter, k6).
- Phối hợp tốt với Developer để reproduce và fix bug nhanh.

## Senior QA Engineer (L4)

**Thâm niên tham khảo**: 4-7 năm.

- Chủ trì test strategy cho 1 sản phẩm hoặc team.
- Xây dựng và maintain automation framework (page object model, data-driven testing).
- Performance testing nâng cao: load test, stress test, capacity planning.
- Security testing cơ bản: OWASP Top 10, SQL injection, XSS.
- Mentor Junior và Mid-level QA.
- Đề xuất cải tiến quy trình testing, giảm time-to-release.
- Tham gia design review và code review (từ góc độ testability).
- Viết tài liệu test strategy, test report cho stakeholders.

## Staff QA Engineer (L5)

**Thâm niên tham khảo**: 7+ năm.

- Ảnh hưởng quality strategy ở mức cross-team / toàn công ty.
- Xây dựng quality culture: "quality là trách nhiệm của tất cả, không chỉ QA".
- Thiết kế test infrastructure: test environments, test data management, CI integration.
- Đánh giá và adopt tools mới (AI-assisted testing, visual regression).
- Định nghĩa quality metrics: bug escape rate, test coverage, MTTR.

## QA Manager (L5 - Management Track)

- Quản lý 4-8 QA engineers.
- Phân bổ resource QA cho các dự án.
- Đảm bảo quality standards nhất quán giữa các team.
- Report quality metrics cho leadership.
- Hiring và phát triển QA team.
